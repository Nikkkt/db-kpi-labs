## 1. Правила переведення слова в число (хеш-код)

### Формат хешу

Хеш-код імені має фіксований формат із трьох частин:

```
[ PPP ][ MMMMMMM ][ LL ]
```

| Частина | Розряди | Опис |
|---------|---------|------|
| `PPP` | 3 цифри | Перші ≤3 літери імені, кожна → цифра 1–9 |
| `MMMMMMM` | 7 цифр | Rolling hash решти літер |
| `LL` | 2 цифри | Довжина слова (01–99) |

**Тип результату:** `long` (64-бітне ціле число)  
**Формула збирання:** `hash = PPP * 10^9 + MMMMMMM * 100 + LL`

---

### Крок 1 — Кодування перших трьох літер (PPP)

Весь алфавіт розбитий на 9 груп. Групи підібрані так, щоб алфавітний порядок букв зберігався у числовому порядку цифр — це дає змогу виконувати діапазонний пошук за алфавітом.

| Група | Українські букви | Англійські букви |
|-------|-----------------|-----------------|
| 1 | А, Б | A, B |
| 2 | В, Г, Д, Е, Є | C, D, E, F |
| 3 | Ж, З, И, І, Ї, Й | G, H, I, J |
| 4 | К, Л | K, L |
| 5 | М, Н, О, Ь | M, N, O |
| 6 | П, Р | P, Q, R |
| 7 | С, Т, У | S, T, U |
| 8 | Ф, Х, Ц, Ч | V, W, X |
| 9 | Ш, Щ, Ю, Я | Y, Z |

Якщо ім'я коротше трьох букв — на відповідну позицію ставиться `0`.

```csharp
long prefix = 0;
for (int i = 0; i < 3; i++)
{
    int g = (i < lower.Length && CharGroup.TryGetValue(lower[i], out int grp)) ? grp : 0;
    prefix = prefix * 10 + g;
}
```

---

### Крок 2 — Rolling hash решти літер (MMMMMMM)

Для символів починаючи з четвертого застосовується polynomial rolling hash:

```
rolling = (rolling × 31 + код_символу) mod 9_999_991
```

Число `9_999_991` — просте, що мінімізує кількість колізій. Результат завжди вкладається в 7 цифр.

```csharp
const long Base = 31L;
const long Mod = 9_999_991L;
long rolling = 0;
for (int i = 3; i < lower.Length; i++)
    rolling = (rolling * Base + lower[i]) % Mod;
```

---

### Крок 3 — Довжина слова (LL)

Довжина імені кодується як двозначне число і додається в кінець хешу. Це дозволяє розрізняти імена з однаковими першими буквами але різною довжиною.

```csharp
long suffix = Math.Min(lower.Length, 99);
return prefix * 1_000_000_000L + rolling * 100L + suffix;
```

### Приклади хеш-кодів

| Ім'я | PPP | MMMMMMM | LL | Хеш |
|------|-----|---------|-----|-----|
| Андрій Захарченко | 111 | 3847291 | 18 | 111384729118 |
| Анна Борченко | 111 | 0092341 | 13 | 111009234113 |
| Зоя Кравченко | 395 | 1837462 | 13 | 395183746213 |
| Михайло Грищенко | 535 | 2947183 | 16 | 535294718316 |
| Заєць | 311 | 0000000 | 05 | 311000000005 |
| Зайченко | 313 | 0044182 | 08 | 313004418208 |
| Зайчатко | 313 | 0039174 | 08 | 313003917408 |

**Спостереження:**
- `"Заєць"` і `"Зайченко"` мають різний PPP (`311` vs `313`), бо третя буква різна — алфавітний порядок збережено.
- `"Зайченко"` і `"Зайчатко"` мають однаковий PPP (`313`), але різний MMMMMMM — вони розрізняються завдяки rolling hash.
- `"Анна..."` і `"Андрій..."` починаються на `111`, але різняться далі — пошук на сторінці пам'яті не потребує додаткових завантажень.

---

### Особливості хеш-функції

1. **Зберігає алфавітний порядок** у старших розрядах — діапазонний пошук числово коректний.
2. **Кодує довжину** — короткі і довгі імена з однаковим початком не конфліктують.
3. **Мінімізує колізії** для різних імен завдяки rolling hash.
4. **Не є обов'язковою** для побудови B+ дерева, але робить range-запити алфавітно коректними без додаткового сортування.

## 2. Опис структури дерева

### Параметри

| Параметр | Значення |
|----------|----------|
| Order (t) | 4 |
| Мінімум ключів у вузлі | 3 (крім кореня — від 1) |
| Максимум ключів у вузлі | 7 |
| Максимальна глибина | 3 |
| Мінімальна ємність | ~24 записи |
| Максимальна ємність | ~448 записів |

**Формула ємності:**
- Мінімум: `2 × (t-1)^(d-1) × (t-1)` де d — глибина
- Максимум: `(2t)^(d-1) × (2t-1)`

---

### Клас BPlusNode

```csharp
public class BPlusNode
{
    public bool IsLeaf;
    public List<long> Keys = new();           // хеш-ключі
    public List<BPlusNode?> Children = new(); // дочірні вузли (тільки внутрішні)
    public List<List<Person>> Records = new(); // записи (тільки листки)
    public BPlusNode? Next;                    // наступний листок (linked list)
}
```

**Ключові правила:**
- Поле `Records` заповнюється **тільки в листках** — посилання на дані зберігаються лише там.
- Поле `Next` утворює **однозв'язний список листків** зліва направо — для ефективного range-пошуку.
- Внутрішні вузли містять лише ключі-роздільники і посилання на дітей.

---

### Клас BPlusTree — методи

```csharp
public class BPlusTree
{
    public const int Order = 4;
    public const int MaxKeys = 7;   // 2*Order - 1
    public const int MinKeys = 3;   // Order - 1
    public const int MaxDepth = 3;
}
```

| Метод | Призначення |
|-------|-------------|
| `Insert(Person)` | Публічна вставка. Обчислює хеш, запускає рекурсію. При розщепленні кореня — створює новий корінь. |
| `InsertRecursive(...)` | Рекурсивний спуск до листка. При переповненні вузла повертає promoted key вгору. |
| `InsertIntoLeaf(...)` | Вставка в листок. При існуючому ключі — додає до слота (колізія/однофамілець). |
| `SplitLeaf(...)` | Розщеплення листка навпіл. Copy-up: перший ключ нового листка копіюється вгору. Оновлює `Next`. |
| `SplitInternal(...)` | Розщеплення внутрішнього вузла. Push-up: середній ключ виштовхується вгору і не залишається в дітях. |
| `Search(string)` | Точний пошук. `FindLeaf` → `BinarySearch` → рядкове порівняння (захист від колізій). |
| `SearchGreaterOrEqual(string)` | Знаходить перший листок і йде по `Next`-linked list. |
| `SearchLess(string)` | Рекурсивний обхід дерева, збирає записи з меншим ключем. |
| `Delete(string)` | Запускає рекурсивне видалення. Якщо корінь спорожнів — замінює його дочірнім. |
| `DeleteRecursive(...)` | Знаходить листок, видаляє запис за іменем. При underflow викликає `FixUnderflow`. |
| `FixUnderflow(...)` | Спочатку позичає у сусіда (`BorrowFromLeft`/`Right`), інакше зливає через `Merge`. |
| `BorrowFromLeft/Right(...)` | Переміщує крайній ключ від сусіда через батьківський роздільник. |
| `Merge(...)` | Зливає два вузли, видаляє роздільник з батька. Для листків оновлює `Next`. |
| `FindLeaf(...)` | Спускається від кореня до листка через `FindChildIndex`. |
| `FindChildIndex(...)` | Лінійним проходом знаходить індекс потрібного дочірнього вузла. |
| `PrintTree()` | Рекурсивно виводить дерево з відступами — тип, ключі, глибину, записи у листках. |

## 4. Схеми пошуку в дереві (обхід)

### 4.1 Точний пошук

**Задача:** знайти `"Іван Франко"`

```
Крок 1. Корінь (глибина 1)
  Keys: [313..., 535..., 767...]
  hash("Іван Франко") = 395...
  395 < 535  →  йдемо в дочірній вузол [1]

Крок 2. Внутрішній вузол (глибина 2)
  Keys: [313..., 395...]
  395 >= 395  →  йдемо в дочірній вузол [1]

Крок 3. Листок (глибина 3)
  Keys: [395..., 411...]
  BinarySearch(395...) → знайдено на позиції 0
  Рядкове порівняння: "Іван Франко" == "Іван Франко" ✓
  Повертаємо запис: +380509012345
```

**Складність:** O(log n) — лише 3 порівняння при глибині 3.

---

### 4.2 Пошук неіснуючого елемента

**Задача:** знайти `"Невідомий Хтось"`

```
Крок 1–3. Спускаємось до листка (аналогічно до точного пошуку)

Крок 3. Листок
  BinarySearch(hash) → індекс від'ємний (~idx)
  Повертаємо порожній список []
```

---

### 4.3 Діапазонний пошук — SearchGreaterOrEqual

**Задача:** знайти всіх, хто йде після `"Михайло Грищенко"` за алфавітом

```
Крок 1. Спускаємось до листка через FindLeaf
  hash("Михайло Грищенко") = 535...
  Знаходимо листок, що містить цей ключ

Крок 2. Збираємо записи з поточного листка
  Keys: [535..., 551...]
  535... >= 535... → додаємо "Леся Тищенко"    ← увага: Леся має менший хеш
  551... >= 535... → додаємо "Михайло Грищенко"

Крок 3. Переходимо по Next до наступного листка
  Keys: [656..., 671...]  → додаємо "Наталія Поліщук", "Олег Сидоренко"

Крок 4. Переходимо по Next далі
  Keys: [767..., 812...]  → додаємо "Петро Власенко", "Роман Савченко"

Крок 5. Next == NULL → зупиняємось
```

**Ключова перевага:** linked list дозволяє пройти діапазон за O(k), де k — кількість результатів, без повернення до кореня.

---

### 4.4 Вставка нового елемента

**Задача:** вставити `"Юрій Петренко"`

```
Крок 1. InsertRecursive спускається до листка
  hash("Юрій Петренко") = 999...
  Йдемо по правій гілці до останнього листка

Крок 2. InsertIntoLeaf
  Keys поточного листка: [767..., 812...]
  999... > 812... → вставляємо в кінець
  Keys стають: [767..., 812..., 999...]  (3 ключі ≤ MaxKeys=7 → OK)

  Розщеплення не потрібне.
```

**Якщо листок переповнений (> 7 ключів):**

```
SplitLeaf:
  Ділимо навпіл: ліва половина залишається, права → новий листок
  Перший ключ нового листка (copy-up) передається батьку
  Оновлюємо Next: старий.Next = новий, новий.Next = старий.Next

  Якщо батько теж переповнився → SplitInternal (push-up вгору)
  Якщо переповнився корінь → створюємо новий корінь, depth++
```

---

### 4.5 Видалення елемента

**Задача:** видалити `"Борис Петренко"`

```
Крок 1. DeleteRecursive спускається до листка
  Знаходимо листок із ключем hash("Борис Петренко")

Крок 2. Видаляємо запис
  Records[idx].RemoveAll(p => p.Name == "Борис Петренко")
  Якщо Records[idx] порожній → видаляємо ключ теж

Крок 3. Перевірка underflow
  Якщо Keys.Count >= MinKeys=3 → все ОК, завершуємо

  Якщо Keys.Count < 3 → FixUnderflow:
    a) Лівий сусід має > 3 ключі → BorrowFromLeft
       Забираємо останній ключ сусіда, оновлюємо роздільник у батьку
    b) Правий сусід має > 3 ключі → BorrowFromRight
       Забираємо перший ключ сусіда, оновлюємо роздільник у батьку
    c) Обидва сусіди мають рівно 3 ключі → Merge
       Зливаємо два листки в один
       Видаляємо роздільник з батька
       Якщо батько теж отримав underflow → повторюємо вгору
```

---

### Порівняння операцій обходу

| | Пошук | Вставка | Видалення |
|---|-------|---------|-----------|
| **Напрямок** | Зверху вниз | Зверху вниз + знизу вгору | Зверху вниз + знизу вгору |
| **Зміна дерева** | Ні | При розщепленні | При злитті / перерозподілі |
| **Складність** | O(log n) | O(log n) | O(log n) |
| **Linked list** | Тільки для range | Оновлення при SplitLeaf | Оновлення при Merge |
| **Найскладніший випадок** | — | Каскадне розщеплення до кореня | Каскадне злиття до кореня |

## Висновки

У роботі реалізовано B+ дерево з хеш-індексом для телефонного довідника. Хеш-функція зберігає алфавітний порядок імен у числовому порядку ключів, що робить діапазонний пошук ефективним і коректним. Зв'язаний список листків дозволяє виконувати range-запити без повернення до кореня. Усі операції — вставка, точний пошук, діапазонний пошук і видалення — працюють за O(log n) і підтримують самобалансування дерева.
